# Connect — Cross-Domain Bridge

Find unexpected connections between two topics by walking the user's
notes and link graph.

## Step 1 — Get the two domains

Ask:

> "Name two topics, projects, or areas you want me to connect.
> The less obviously related, the better."

Example pairs: "investing" + "parenting", "writing" + "engineering",
"health" + "hiring"

## Step 2 — Read both domains

For each domain:

1. Grep the vault for mentions of the topic
2. Read matching notes fully
3. Track key concepts, metaphors, principles, and frameworks
   the user uses in each domain

## Step 3 — Walk the bridge

Find connections:

- **Shared principles** — same underlying idea expressed differently
  in each domain
- **Transferable frameworks** — a mental model from one domain that
  applies to the other
- **People bridges** — individuals who appear in both contexts
- **Tension points** — where the two domains seem to contradict,
  which often reveals the most interesting insight

## Step 4 — Present the bridge

```
## Connecting: [Domain A] ↔ [Domain B]

### Shared ground
- [Principles or patterns that appear in both]

### Frameworks that transfer
- [A mental model from A that illuminates B, or vice versa]

### Surprising tensions
- [Where the two domains contradict — and what that reveals]

### The bridge insight
- [One synthesized idea that only emerges from holding both together]
```

## Rules

- **Read only.** Do not create, edit, or move any vault files.
- Build connections from the user's own notes, not generic knowledge.
- If one domain has thin coverage in the vault, say so and work with
  what's there.
- Prioritize surprising connections over obvious ones.
