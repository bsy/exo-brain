# Close — End of Day Review

Review the day's work and surface what matters before you sign off.

## Step 1 — Read today's context

Read in parallel:

1. `daily/[today's date].md` — the day's note
2. Any files in `inbox/` that arrived today
3. Any project files modified today (check git status or file timestamps)

## Step 2 — Generate the review

Present a structured end-of-day review:

```
## Day in Review — [date]

### What got done
- [Completed work, decisions made, progress on projects]

### Action items captured
- [ ] [Extracted from today's notes — things mentioned but not yet done]

### Missed connections
- [People mentioned who might need follow-up]
- [Ideas that connect to existing projects or research]
- [Threads left dangling]

### Tomorrow's seed
- [1-2 suggested focus areas based on today's momentum]
```

## Step 3 — Ask, don't write

Present the review and ask:

> "Anything you want to add to today's note before closing out?"

If they say yes, tell them what to add — but let them write it.

## Rules

- **Read only.** Do not create, edit, or move any vault files.
- Be honest if the day's notes are thin — don't fabricate substance.
- Keep it short. This is a wind-down, not a debrief.
