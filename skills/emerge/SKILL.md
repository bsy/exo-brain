# Emerge — Pattern Discovery

Scan the vault to find ideas, themes, and connections that are implied
by the user's notes but never stated directly.

## Step 1 — Scan broadly

Read across the vault to build a picture:

1. `daily/` — last 14 daily notes (or all if fewer exist)
2. `projects/` — index/README of each active project
3. `research/` — all files
4. `inbox/` — anything unprocessed

Track recurring themes: words, people, topics, questions, frustrations,
and aspirations that appear across multiple notes.

## Step 2 — Find what's implicit

Look for:

- **Recurring concerns** — topics the user keeps circling back to
  without resolving
- **Unstated connections** — two projects or ideas that share a thread
  the user hasn't linked
- **Emerging priorities** — what's gaining energy in recent notes vs.
  what's fading
- **Unasked questions** — things their notes imply they're wondering
  about but haven't articulated

## Step 3 — Present discoveries

```
## Emergent Patterns

### Threads you keep pulling
- [Recurring themes with evidence from specific notes]

### Connections hiding in plain sight
- [Two+ notes/projects that share an unstated link]

### What's gaining energy
- [Topics increasing in frequency or depth recently]

### What your notes are asking (even if you haven't)
- [Implicit questions or curiosities surfaced from the material]
```

## Step 4 — Invite reflection

> "Any of these resonate? I can dig deeper into any pattern,
> or run /connect to explore a specific link."

## Rules

- **Read only.** Do not create, edit, or move any vault files.
- Only surface patterns from the user's own notes — never inject
  external ideas or AI-generated theories.
- Be specific: cite which notes contribute to each pattern.
- If the vault is too sparse for meaningful patterns, say so honestly.
